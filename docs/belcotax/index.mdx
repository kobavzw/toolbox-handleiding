## Validatiemodule inkomstenjaar 2024 

Om de aangemaakte XML-bestanden om te zetten naar .BOW bestanden die je kan uploaden in Belcotax, moet je de validatiemodule van Belcotax gebruiken.
Die kan je hier downloaden:
https://financien.belgium.be/nl/E-services/Belcotaxonweb/technische-documentatie

Of via deze rechtstreekse link: https://ccff02.minfin.fgov.be/CCFF_SP7_2015/jnlp/belcotax.jnlp

Heb je hulp nodig om deze validatiemodule te installeren of op te starten, bekijk dan zeker deze pagina:
https://financien.belgium.be/nl/E-services/Belcotaxonweb/technische-documentatie/validatiemodule-handleiding


## XML

Mogelijk krijg je bij het valideren van een XML bestand foutmeldingen. Als het gaat om **niet-blokkerende foutmeldingen** mag je die negeren en kan het bestand
alsnog correct worden ingediend.

Als het gaat om **blokkerende foutmeldingen**, dan moet je actie ondernemen en het achterliggende probleem proberen oplossen.

In de XML staat alles op één lijn wat niet makkelijk leesbaar is. Er is een relatief makkelijke manier om de XML netjes te formatteren zodat die (ook weer relatief) leesbaar wordt. Hiervoor installeer je het gratis Visual Studio Code van Microsoft: https://code.visualstudio.com/download
 
Eens opgestart kan je links op de vier blokjes klikken (extensies). 

Daar zoek je naar de extensie **XML Tools** en je installeert die. 

Als je nu het XML bestand met Visual Studio Code opent kan je in het **menu View kiezen voor Command palette**. 
Klik daar op **XML: format as XML**. De XML wordt nu netjes weergegeven. 

Als je dat bestand bewaart en terug door de validatie laat gaan, krijg je duidelijke regelnummers te zien en kan je makkelijker vaststellen waar het probleem juist zit.
